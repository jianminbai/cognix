# Spring Cloud 系列课程 · 第 7 课

**进度：7/30**

## Sentinel 进阶：流控规则、降级规则与热点规则

### 🎯 学习目标
- 掌握 Sentinel 三种流控模式（直接/关联/链路）与流控效果（快速失败/Warm Up/排队等待）
- 理解熔断降级的三种策略：慢调用比例、异常比例、异常数
- 学会配置热点参数限流与参数例外项
- 能够结合业务场景选择合适的规则组合

### 📖 核心内容

#### 1. 流控规则（FlowRule）

**流控模式：**

| 模式 | 说明 | 适用场景 |
|------|------|---------|
| 直接 | 对资源自身限流 | 最常见的单接口限流 |
| 关联 | 当关联资源达到阈值时，对当前资源限流 | 支付接口降级时限制订单接口流量 |
| 链路 | 根据调用链入口限流 | 区分用户端和管理端的同一方法调用 |

**流控效果：**

| 效果 | 说明 |
|------|------|
| 快速失败 | 直接抛出 FlowException，默认行为 |
| Warm Up | 预热模式，阈值从 coldFactor（默认3）逐步升至设定值 |
| 排队等待 | 匀速排队让请求匀速通过，需设置超时时间 |

**代码示例：**

```java
@GetMapping("/order/create")
@SentinelResource(value = "order:create", 
                  fallback = "orderFallback")
public Result createOrder(@RequestParam Long userId) {
    return orderService.create(userId);
}
```

配置规则：资源名 = `order:create`，流控模式 = `关联`，关联资源 = `pay:create`，阈值 = `50 QPS`。当支付接口 QPS >= 50 时，订单接口自动被限流。

#### 2. 熔断降级规则（DegradeRule）

**三种降级策略：**

| 策略 | 说明 | 阈值类型 | 典型配置 |
|------|------|---------|---------|
| 慢调用比例 (SLOW_REQUEST_RATIO) | 响应时间超过 maxRt 算一次慢调用，比例超阈值则熔断 | 比例（0~1） | RT > 500ms 比例 > 0.5 则熔断 |
| 异常比例 (ERROR_RATIO) | 异常数 / 总调用数 > 阈值则熔断 | 比例（0~1） | 异常比例 > 20% 则熔断 |
| 异常数 (ERROR_COUNT) | 一分钟内异常数 > 阈值则熔断 | 整数 | 一分钟内 5 次异常则熔断 |

**熔断器状态机：**
- **CLOSED**（关闭）：正常状态，请求正常通过
- **OPEN**（开启）：熔断触发，请求直接降级 fallback
- **HALF_OPEN**（半开）：放行少量请求探测服务是否恢复

#### 3. 热点参数限流（ParamFlowRule）

热点规则针对**携带特定参数值**的请求进行精细化限流。例如：秒杀场景中热门商品 ID 的流量明显高于普通商品。

```java
@GetMapping("/product/{id}")
@SentinelResource(value = "product:getDetail")
public Result getProductDetail(@PathVariable Long id) {
    // 普通商品限流 100 QPS，商品 9527 限流 500 QPS
    return productService.getDetail(id);
}
```

配置：对第1个参数限流，默认阈值100 QPS，参数例外项：id=9527 → 500 QPS。

#### 4. 三种规则对比与选型

| 规则类别 | 解决问题 | 推荐场景 |
|---------|---------|---------|
| 流控规则 | 流量过载保护 | 高峰期削峰、接口限流、防止爬虫 |
| 降级规则 | 服务容错保护 | 第三方接口不稳定、慢调用、数据库抖动 |
| 热点规则 | 参数级精细限流 | 秒杀、热点商品、热搜词 |

**组合原则：**
1. 流控 + 降级搭配使用：流控管量，降级管质
2. 热点规则优先：先识别热点再兜底流控
3. TimeWindow 不宜太短（<5s 无意义），也不宜太长（>60s 影响恢复）

### 💡 最佳实践
1. **避免规则冲突**：同时对同一资源配置流控和降级规则时，确保阈值窗口错开
2. **慢调用降级的 maxRt**：留 30%-50% 余量，避免网络抖动误熔断
3. **资源命名规范**：`service:method` 格式，统一前缀便于管理
4. **规则持久化**：生产环境不要依赖控制台推送，用 Nacos 数据源（见 L8）
